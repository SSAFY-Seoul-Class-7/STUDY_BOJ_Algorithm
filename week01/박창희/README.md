# [백준 1931번 - 회의실 배정](https://www.acmicpc.net/problem/1931)

### 💡 핵심 아이디어 & 알고리즘
- **알고리즘:** Greedy (그리디), 정렬 (Sorting)
- **시간 복잡도:** O(N log N) (N=100,000 이므로 O($N^2$) 불가)
    PriorityQueue의 add / poll -> 하나당 $O(\log N)$

- **접근 방법:**
    1. 회의를 최대한 많이 배정하려면 "가장 빨리 끝나는 회의"를 먼저 선택해야 한다.
    2. 따라서 종료 시간을 기준으로 오름차순 정렬한다.

# [백준 1939번 - 중량제한](https://www.acmicpc.net/problem/1939)

### 💡 핵심 아이디어 & 알고리즘

- **알고리즘:** 이분 탐색 (Binary Search), 너비 우선 탐색 (BFS)
- **시간 복잡도:** O((N + M) log C)
  - **이분 탐색:** 가능한 중량 범위(1 ~ 10억)를 절반씩 줄여가며 탐색하므로 **O(log C)** 회 반복합니다. (C: 중량의 최댓값)
  - **BFS:** 특정 중량 이상으로 이동 가능한지 확인하기 위해 그래프를 탐색하므로 **O(N + M)** 시간이 소요됩니다. (N: 섬의 수, M: 다리의 수)
    -> 인접리스트로 할 경우 O(V+E) 구조
  - **총 소요 시간:** 이분 탐색(log C) \* BFS(N + M) = **O((N + M) log C)**
- **접근 방법:**
  1. 중량의 최댓값(C)이 10억이므로 가능한 중량을 1부터 MAX까지 **이분 탐색**으로 결정합니다.
  2. 결정된 중량(mid)으로 시작점에서 도착점까지 이동할 수 있는지 **BFS**로 확인합니다.
  3. 이동 가능하다면 더 큰 중량을 찾기 위해 `low = mid + 1`로 범위를 좁히고, 불가능하다면 `high = mid - 1`로 범위를 좁힙니다.
  4. 초기 `high` 값을 입력받은 중량 중 최댓값(`MAX`)으로 설정하여 탐색 범위를 최적화했습니다.
